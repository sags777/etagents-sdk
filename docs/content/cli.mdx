# CLI Reference

The `eta` binary is installed when you add `@etagents/sdk`. It provides 12 commands for running agents, managing sessions, working with vector memory, and tooling tasks.

```bash
npx eta --help
```

---

## eta run

Execute an agent file against a single prompt.

```bash
eta run ./my-agent.ts "What is the capital of France?"
eta run ./my-agent.ts "Summarise Q4 sales" --model claude-sonnet-4-6
eta run ./my-agent.ts "Hello" --session-id user-alice --events --show-usage
```

| Flag | Description |
|---|---|
| `--model <id>` | Override the model bound on the agent |
| `--api-key <key>` | API key used with `--model` |
| `--session-id <id>` | Session run ID for continuity |
| `--max-turns <n>` | Maximum turns override |
| `--show-usage` | Print usage totals to stderr |
| `--show-turns` | Print final turn count to stderr |
| `--show-tool-calls` | Print tool call names to stderr |
| `--events` | Stream run events to stderr |
| `--json` | Emit the full structured `RunResult` |

Your agent file must export a default `AgentDef`:

```typescript title="my-agent.ts"
import { createAgent } from "@etagents/sdk";
export default createAgent({ name: "assistant", systemPrompt: "...", model: "claude-sonnet-4-6" });
```

## eta chat

Start an interactive REPL session with an agent.

```bash
eta chat ./my-agent.ts
eta chat ./my-agent.ts --session-id user-alice --model claude-sonnet-4-6
```

Type a message and press Enter to send. Submit an empty line or press `Ctrl+C` to quit. The conversation persists if `--session-id` is set.

---

## eta exec

Execute a single tool from an agent file directly, bypassing the LLM.

```bash
eta exec ./my-agent.ts search_database --args '{"query":"Q4 sales","limit":5}'
```

Useful for testing tool handlers in isolation without spinning up a full agent run.

---

## eta session

Manage persisted session history.

```bash
# List all sessions in the store
eta session list --store file:.sessions

# Inspect a specific session
eta session get --session-id user-alice --store file:.sessions

# Delete a session
eta session delete --session-id user-alice --store file:.sessions
```

`--store` accepts `file:<dir>` or `redis:<url>`. `eta session list` prints discovered run IDs; `--json` returns the raw list or snapshot.

---

## eta memory

Work with vector memory (requires `RedisMemory` provider).

```bash
# Index a session's transcript into memory
eta memory index --session-id user-alice --store file:.sessions

# Search memory for relevant context
eta memory search "user preferences" --top-k 5
```

| Flag | Description |
|---|---|
| `--redis-url <url>` | Redis URL for `RedisMemory` |
| `--embedder-key <key>` | OpenAI API key for embeddings |
| `--namespace <ns>` | Memory namespace |
| `--top-k <n>` | Number of results to return (default: 5) |
| `--min-score <n>` | Minimum similarity score (0\u20131) |
| `--ttl-days <n>` | TTL in days for indexed entries |
| `--session-id <id>` | Session ID used by `index` |
| `--store <spec>` | Session store spec for `index` |
| `--json` | Emit raw search results |

## eta scan

Discover agent files and MCP server configs in a directory.

```bash
eta scan ./src
eta scan ./src --mcp   # also show *.mcp.json files
```

Outputs a list of discovered `*.agent.ts`, `*.agent.js`, and (with `--mcp`) `*.mcp.json` files. Skips `node_modules` and dotfile directories.

See the [Scanner guide](./guides/advanced/scanner) for programmatic usage.

---

## eta inspect

Display the structure of an agent: manifest fields, tools, and MCP servers.

```bash
eta inspect ./my-agent.ts
eta inspect ./my-agent.ts --session-id user-alice --json
```

Outputs the agent's name, optional description and version, model class, system prompt, tool names, and MCP servers. `--json` includes tool schemas and optional session data.

---

## eta build

Validate an agent file and optionally emit its manifest.

```bash
eta build ./my-agent.ts
eta build ./my-agent.ts --out ./agent.manifest.json
eta build ./my-agent.ts --typecheck-only
```

Loads the exported default `AgentDef` and writes a manifest that includes description, version, model class, tools, and MCP config. Use `--typecheck-only` when you only want validation.

---

## eta serve

Expose an agent's tools as an MCP server over stdio. Other agents can connect to it using `transport: "stdio"`.

```bash
eta serve ./my-agent.ts
```

The server stays running until you send `Ctrl+C`. Connect from another agent:

```typescript
const agent = createAgent({
  mcp: [{ transport: "stdio", command: "eta", args: ["serve", "./my-agent.ts"], serverName: "my-server" }],
});
```

---

## eta orchestrate

Route a prompt through an explicit pool of agent files using `AgentRouter`.

```bash
eta orchestrate "Research and summarise AI trends" --agents ./planner.agent.ts,./writer.agent.ts
```

| Flag | Description |
|---|---|
| `--agents <paths>` | Comma-separated agent file paths |
| `--router <name>` | Routing strategy. Currently `triage` is supported |
| `--triage-model <id>` | Override the model used for triage |
| `--triage-api-key <key>` | API key for the triage model override |
| `--session-id <id>` | Run ID for continuity |
| `--show-usage` | Print usage totals |
| `--json` | Emit the full structured result |

---

## eta init

Scaffold a new agent file.

```bash
eta init my-agent
eta init my-agent --with-tools --with-mcp
```

Creates `my-agent.agent.ts` in the current directory. Optional flags add a sample tool scaffold and MCP client boilerplate.

---

## eta mask

Test PII masking rules on a string.

```bash
eta mask "My email is alice@example.com and my SSN is 123-45-6789" --rules email,ssn
```

Outputs the masked string with placeholders, and lists the detected matches. Useful for verifying that your privacy rules cover the expected patterns.

| Flag | Description |
|---|---|
| `--rules <list>` | Comma-separated built-in rules: `email`, `phone`, `ssn`, `creditCard`, `ipAddress`, `name` |
| `--pattern <regex>` | Append a custom regex rule |
| `--passphrase <key>` | Encrypt the privacy map at rest |
| `--json` | Emit masked text plus the privacy map |
