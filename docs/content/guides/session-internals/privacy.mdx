# Privacy & PII Redaction

`RegexPrivacy` masks personally identifiable information before every LLM call and automatically restores it in the model's response. Your tool handlers always receive the original unmasked values.

```typescript
import { createAgent, startRun, RegexPrivacy, BUILTIN_RULES } from "@etagents/sdk";

const agent = createAgent({
  name: "support-bot",
  systemPrompt: "You are a support agent. Acknowledge the user's request.",
  model: "claude-sonnet-4-6",
  privacy: new RegexPrivacy({
    rules: [BUILTIN_RULES.email, BUILTIN_RULES.phone],
  }),
});

// Input contains PII
const result = await startRun(
  agent,
  "My email is alice@example.com and phone is +1-555-867-5309",
);

// The LLM never saw the real email or phone — it worked with \u27e8eta:email:0001\u27e9 and \u27e8eta:phone:0001\u27e9
// result.response contains the unmasked values restored automatically
console.log(result.response);
```

Placeholder format: `⟨eta:email:0001⟩` — unambiguously distinct from any user-generated text.

## How it works

```
User input arrives
   ↓
PrivacyFence.mask()  ← regex rules applied, map built: \u27e8eta:email:0001\u27e9 → alice@example.com
   ↓
Masked input sent to LLM
   ↓
LLM responds with masked placeholders
   ↓
PrivacyFence.unmask()  ← map applied in reverse: \u27e8eta:email:0001\u27e9 → alice@example.com
   ↓
Unmasked response returned in RunResult
```

Tool handlers receive unmasked arguments. If the LLM passes `⟨eta:email:0001⟩` as a tool argument, the kernel unmasks it before calling your handler so your handler always receives `alice@example.com`.

The `PrivacyFence` accumulates the replacement map across all turns in a run — placeholders first introduced in turn 1 are correctly unmasked in turn 5.

## Built-in rules

```typescript
import { BUILTIN_RULES } from "@etagents/sdk";

new RegexPrivacy({
  rules: [
    BUILTIN_RULES.email,       // alice@example.com → \u27e8eta:email:0001\u27e9
    BUILTIN_RULES.phone,       // +1-555-867-5309 → \u27e8eta:phone:0001\u27e9
    BUILTIN_RULES.ssn,         // 123-45-6789 → \u27e8eta:ssn:0001\u27e9
    BUILTIN_RULES.creditCard,  // 4111-1111-1111-1111 → \u27e8eta:cc:0001\u27e9
    BUILTIN_RULES.ipv4,        // 192.168.1.1 → \u27e8eta:ip:0001\u27e9
  ],
});
```

| Rule | Matches |
|---|---|
| `BUILTIN_RULES.email` | RFC 5321 email addresses |
| `BUILTIN_RULES.phone` | US and international phone numbers |
| `BUILTIN_RULES.ssn` | US Social Security Numbers (`\u03a7\u03a7\u03a7-\u03a7\u03a7-\u03a7\u03a7\u03a7\u03a7`) |
| `BUILTIN_RULES.creditCard` | Visa, Mastercard, Amex, Discover formats |
| `BUILTIN_RULES.ipv4` | IPv4 addresses (`192.168.x.x`, `10.x.x.x`, etc.) |

## Custom patterns

Add domain-specific identifiers via `customRules`:

```typescript
new RegexPrivacy({
  rules: [BUILTIN_RULES.email, BUILTIN_RULES.phone],
  customRules: [
    { pattern: /ACCT-\d{8}/g, label: "account" },
    { pattern: /EMP-[A-Z]{3}-\d{4}/g, label: "employee" },
  ],
});
```

Custom rule patterns must have the `g` flag. The `label` becomes part of the placeholder: `ACCT-12345678` → `⟨eta:account:0001⟩`.

## Standalone masking

You can use `RegexPrivacy` outside of an agent run for preprocessing or testing:

```typescript
import { RegexPrivacy, BUILTIN_RULES } from "@etagents/sdk";

const privacy = new RegexPrivacy({
  rules: [BUILTIN_RULES.email, BUILTIN_RULES.phone],
});

// Mask
const { masked, map } = await privacy.mask(
  "Contact alice@example.com or call +1-555-867-5309",
);
console.log(masked);
// "Contact \u27e8eta:email:0001\u27e9 or call \u27e8eta:phone:0001\u27e9"

// Unmask
const original = await privacy.unmask(masked, map);
console.log(original);
// "Contact alice@example.com or call +1-555-867-5309"
```

## Privacy map storage

The `PrivacyFence` does not persist the privacy map. If you need HITL + privacy together (the run suspends, the map must survive a process restart), configure a `hitlStore` on `HitlConfig` — the kernel includes the privacy map in the `SuspendSnapshot`. See the [HITL guide](../advanced/hitl).

## Implementing a custom PrivacyProvider

If `RegexPrivacy` is not sufficient (e.g. you need NLP-based NER), implement the `PrivacyProvider` interface:

```typescript
import type { PrivacyProvider, PrivacyMap, MaskResult } from "@etagents/sdk";

class NerPrivacy implements PrivacyProvider {
  async mask(text: string): Promise<MaskResult> {
    // Call your NER model, build the map
    const map: PrivacyMap = new Map();
    const masked = text.replace(/* ... */, (match) => {
      const id = generateId();
      map.set(id, match);
      return id;
    });
    return { masked, map };
  }

  async unmask(text: string, map: PrivacyMap): Promise<string> {
    let result = text;
    for (const [placeholder, original] of map) {
      result = result.replaceAll(placeholder, original);
    }
    return result;
  }
}
```
