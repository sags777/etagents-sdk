# Lifecycle Hooks

Lifecycle hooks let you inject logic at key moments in a run without modifying the kernel or your tools. Use them for telemetry, audit logging, and debugging.

```typescript
import { createAgent } from "@etagents/sdk";

const agent = createAgent({
  name: "assistant",
  systemPrompt: "...",
  model: "claude-sonnet-4-6",
  hooks: {
    onTurnStart: async (turn, context) => {
      console.log(`Turn ${turn}: sending to model`);
    },
    onToolCall: async (call, context) => {
      await telemetry.record("tool_call", { name: call.name, runId: context.runId });
    },
    onToolResult: async (result, context) => {
      await telemetry.record("tool_result", { id: result.toolCallId, error: result.isError });
    },
    beforeComplete: async (messages, context) => {
      await auditLog.write(messages);
    },
  },
});
```

All hooks are **fail-open**: if a hook throws, the error is logged and the run continues. Hooks can never crash a session.

---

## Available hooks

| Hook | Signature | When |
|---|---|---|
| `onTurnStart` | `(turn: number, context: HookContext) => void \| Promise<void>` | Before each LLM call |
| `onTurnEnd` | `(turn: number, context: HookContext) => void \| Promise<void>` | After each LLM response |
| `onToolCall` | `(call: ToolCall, context: HookContext) => void \| Promise<void>` | Before each tool executes |
| `onToolResult` | `(result: ToolResult, context: HookContext) => void \| Promise<void>` | After each tool returns |
| `beforeComplete` | `(messages: Message[], context: HookContext) => void \| Promise<void>` | Just before the run returns |

All hooks may be async. The kernel awaits each hook before continuing.

---

## onTurnStart / onTurnEnd

Use these to instrument LLM call latency:

```typescript
const turnStartTimes = new Map<number, number>();

const agent = createAgent({
  ...,
  hooks: {
    onTurnStart: async (turn) => {
      turnStartTimes.set(turn, Date.now());
    },
    onTurnEnd: async (turn, context) => {
      const start = turnStartTimes.get(turn);
      if (start) {
        const latencyMs = Date.now() - start;
        await metrics.record("llm_latency", { turn, latencyMs });
      }
    },
  },
});
```

---

## onToolCall / onToolResult

Use these to audit every tool invocation:

```typescript
hooks: {
  onToolCall: async (call, context) => {
    // call.name, call.args, call.id
    console.log(`Calling ${call.name} with`, call.args);
  },
  onToolResult: async (result, context) => {
    // result.toolCallId, result.result (string), result.isError, result.durationMs
    if (result.isError) {
      await alerting.trigger("tool_error", { id: result.toolCallId });
    }
  },
},
```

---

## beforeComplete

`beforeComplete` receives the full `Message[]` array just before the run returns. Use it for final audit logging or message archiving:

```typescript
hooks: {
  beforeComplete: async (messages, context) => {
    // Full conversation including tool calls and results
    await db.archiveConversation(context.runId, messages);
  },
},
```

:::note
`beforeComplete` fires for every run exit status — `"complete"`, `"error"`, `"cancelled"`, and `"budget_exceeded"`. Check the `RunResult` status in your `onEvent` callback if you only want to act on successful completions.
:::

---

## Hooks vs onEvent

| | Hooks | `onEvent` |
|---|---|---|
| Defined on | `AgentConfig` (applies to all runs) | `RunConfig` (per-run) |
| Error handling | Fail-open (never crashes) | Fail-open |
| Access to run context | Yes (`agentName`, `runId`, `turn`) | Depends on the event payload |
| Use for | Persistent telemetry, audit logging | Per-run UI updates, progress display |

For simple per-run observation, prefer `onEvent`. Reserve hooks for cross-cutting concerns you want on every run using this agent.
